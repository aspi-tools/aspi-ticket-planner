# Ticket Planner 使用マニュアル

スタッフ向け操作ガイド（v0.4 / 2026-05-07）

---

## 1. 起動方法

### iPad
1. ホーム画面のQRリーダー（カメラ）で `運用QR.png` を読み取り
2. Safariで `https://aspi-tools.github.io/aspi-ticket-planner/` が開く
3. 「ホーム画面に追加」しておくと次回からアイコンタップで起動

### PC・Mac
ブラウザで上記URLを直接開く

---

## 2. 画面構成

ヘッダー下に2つのタブ:

| タブ | いつ使う |
|---|---|
| **消化プラン** | 残チケットの消化計画を組みたいとき（ダウンセル提案・回帰提案・通うペース提案） |
| **チケットの価値** | 退会希望のお客様に残チケットの金額インパクトを伝えたいとき |

---

## 3. 「消化プラン」の使い方

### 入力項目
| 項目 | 説明 |
|---|---|
| 現プラン | お客様の現在のプラン（月N回） |
| 残チケット | 現在の残量。「分」または「回」で入力（1回=30分） |
| 手続き適用月 | プラン切替を始める年月 |
| 月あたり通うペース | お客様が約束できる通う頻度。分または回で入力 |
| 手続き後プラン候補 | 比較したいプランを最大3つタップ選択 |

入力した瞬間に下のグラフ・カードが更新されます（計算ボタン不要）。

### 出力の見方
- **タイムラインチャート**: 各候補プランごとに残チケットが減っていく曲線。線が0にぶつかる月＝消化完了月
- **カード（候補ごと）**:
  - **○年○月** ← 使い切れる年月（顧客に見せるメイン情報）
  - **予約上限 ◯件** ← そのプランで同時保持できる予約数
  - 月◯分（◯回）の純消化／全◯ヶ月 ← スタッフ補足用
- **「計画的に減らせません」表示**: 月の付与回数 ≧ 通うペース のとき出る。下位プラン or ペース増を提案

### トーク例
> 「いま残りが1080分（36回）あります。月8プランのまま月8回ペースで通えば、ちょうど来年4月までに使い切れる計算です。月4プランに下げると『使い切る』までは伸びますが、無理なくお通いいただけますね」

---

## 4. 「チケットの価値」の使い方

### 入力項目
| 項目 | 説明 |
|---|---|
| 会費区分 | 通常 / 旧〜2025 / 旧〜2022 / 特別 のうち該当 |
| 現プラン | お客様の現プラン |
| 残チケット | 現在の残量 |

### 出力の見方
- 大きな金額 ← 残チケットの**金額換算**（会費区分×プランから動的算出）
- 「月会費の約 N.Nヶ月分に相当」 ← 月会費何ヶ月分か
- 「内部単価: ¥X/分」 ← スタッフ参照用（顧客には説明不要）
- 「**この残量で消化計画を見る →**」ボタン ← 残量を引き継いで消化プランへ遷移

### トーク例
> 「いま退会されると、残っている1080分（36回）のチケットがそのまま消滅してしまいます。月会費の約4.5ヶ月分にあたる金額です。せっかく持っていらっしゃるので、月◯ペースなら◯ヶ月で使い切れます。ご一緒に計画を見てみませんか」

→ そのまま「消化計画を見る」ボタンで Mode A へ遷移、続けて消化プランを提案

## 5. 注意事項

### 計算ロジックの前提
- 計算式は Google Sheets「退会＆ダウンセル阻止ツール」を踏襲
- 1回 = 30分換算
- 消化月数 = 切上げ（端数月も1ヶ月とカウント）

### データの最新性
- 単価・予約上限: 『各種サービス』PDF (2026-03-31) ベース
- 2026年1月会費改定後の単価
- **月14×特別会費・月48全区分・月60全区分は料金未登録** → 選択時に「料金データなし」表示
- 料金変更があれば `docs/data/pricing.json` を更新してGitHubにpush

### 顧客に見せない情報
- バッジの「内部単価」表示は会話中に強調しない
- 入力欄の「会費区分」も大々的には触れない（特に「特別会費」はスタッフ家族向けなのでお客様には見せない）

### 予約上限の特例ルール
- 回数変更切替時は「12日（翌月枠公開日）時点で予約上限の多い方が適用」
- アプリでは候補プランの上限のみ表示。切替時のリアル予約数はASPI Smile/デジムで確認

### ブラウザ
- iPad Safari推奨（Chart.jsの描画が綺麗）
- ネット接続必須（Tailwind/Chart.jsをCDNから読み込み）

## 6. トラブルシュート

| 症状 | 対処 |
|---|---|
| 開かない / 真っ白 | ネット接続を確認。CDN（cdn.tailwindcss.com / cdn.jsdelivr.net）が必要 |
| 「料金データなし」表示 | 月14×特別 or 月48・月60を選んだ場合の正常動作。プラン変更で対処 |
| 数字が大きすぎ/小さすぎ | 「分」「回」の単位選択を確認。1回=30分 |
| 古い情報が出る | Safariのリロード（アドレス欄を上から下にスワイプ） |
